# E5: Alert Pipeline Strategy Comparison

## Overview

Compares two strategies for evaluating ATC safety rules against projected traffic states:

- **compiled** ‚Äî rules generated by E4 (`.py` files with `evaluate(ts, callsign)`)
- **generic** ‚Äî LLM at runtime evaluating each rule via prompt (Instructor `MD_JSON`)

Each strategy is scored on **alert accuracy** (precision, recall, F1, severity accuracy), **semantic quality** (LLM judge), and **latency**.

## What Gets Measured

Each strategy runs all 6 test cases. Per test case, per rule:

| Metric | Description |
|---|---|
| **Alert Precision** | TP / (TP + FP) ‚Äî false alarm rate |
| **Alert Recall** | TP / (TP + FN) ‚Äî missed violations |
| **Alert F1** | Harmonic mean of precision & recall |
| **Severity Accuracy** | Fraction of violations where severity matches expected |
| **Semantic Judge Score** | LLM holistic score (0.0‚Äì1.0) per test case |
| **Latency** | Avg execution time per rule (ms) |

### Confusion Matrix

`satisfied = True` ‚Üí safe (no alert). `satisfied = False` ‚Üí violation (alert).

| | Predicted Violation | Predicted Safe |
|---|---|---|
| **Expected Violation** | True Positive | False Negative |
| **Expected Safe** | False Positive | True Negative |

### Overall Score

Weighted combination:
- Precision: 30% / Recall: 30% / Severity accuracy: 20% / Semantic quality: 20%

## Requirements

- **Compiled rules**: `E4_compiled_rules/models/model_A(gemma4:e4b)/` (configurable with `--compiled-rules-dir`)
- **LLM model**: `gemma4:31b-cloud` via Ollama (default, configurable with `--judge-model` / `--generic-model`)
- **Python deps**: `instructor`, `pydantic`, `matplotlib`, `numpy`

## Usage

```bash
# Full run (default: gemma4:31b-cloud via Ollama)
uv run python experiments/E5_alert_pipeline/src/run.py

# Skip semantic judge (faster)
uv run python experiments/E5_alert_pipeline/src/run.py --no-judge

# Custom models for judge and generic evaluator
uv run python experiments/E5_alert_pipeline/src/run.py \
    --judge-model gemma4:31b-cloud \
    --generic-model gemma4:31b-cloud

# Custom output dir
uv run python experiments/E5_alert_pipeline/src/run.py \
    --output /tmp/e5_results
```

## Key Differences from E6

| Aspect | E5 | E6 |
|---|---|---|
| Scope | Strategy comparison only | Full system latency & accuracy |
| Pipeline | Direct rule evaluation | BERT + rules + full AlertPipeline |
| Test cases | 6 (all projected states) | 8 (native+compiled + generic paths) |
| Iterations | Single measure | Warmup + 15 measures |
| Focus | Which strategy is more accurate? | How fast is each component? |
